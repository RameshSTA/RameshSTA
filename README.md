### Argus — Insurance Intelligence Platform

<div align="center">

[![Live Demo](https://img.shields.io/badge/Live_Demo-Visit-009688?style=for-the-badge)](https://RameshSTA.github.io/argus)&nbsp;
[![API Docs](https://img.shields.io/badge/API_Docs-OpenAPI-FF6B35?style=for-the-badge)](https://Ramesh79-argus.hf.space/api/docs)&nbsp;
[![Repository](https://img.shields.io/badge/GitHub-Repository-181717?style=for-the-badge)](https://github.com/RameshSTA/argus)

</div>

<br/>

A **production-deployed** three-layer AI platform for insurance intelligence — combining machine learning fraud detection, retrieval-augmented generation, and agentic claims triage into a single live system. The FastAPI backend runs on Hugging Face Spaces with full OpenAPI documentation; the frontend is continuously deployed to GitHub Pages via GitHub Actions CI/CD on every push to `main`.

Designed and grounded in primary-source research into **Suncorp Group's FY2024–FY2025 AI transformation**: SunGPT on Databricks Mosaic AI, 14,350+ staff hours saved, 2M+ AI-generated claims summaries, and a five-year Microsoft Azure strategic partnership — each of Argus's three capability layers maps directly to a verified Suncorp production system.

<br/>

<div align="center">

| Metric | Value | Context |
|:---|:---:|:---|
| Fraud Detection AUC-ROC | **99.8%** | XGBoost on 555K synthetic claims |
| Policy Documents Indexed | **847** | FAISS vector store, Sentence-Transformers |
| AI Capability Layers | **3** | Fraud · RAG · Agentic |
| API Response Latency | **< 200 ms** | FastAPI on Hugging Face Spaces |
| Deployment Status | **Live** | GitHub Pages + Hugging Face Spaces |

</div>

<br/>

**Layer 1 — Fraud Detection Engine**

XGBoost classifier trained on 555,000 synthetic insurance claims with SMOTE resampling to address class imbalance and 47 domain-specific engineered features. SHAP waterfall plots surface per-claim risk drivers for every prediction, producing a fully auditable decision trail aligned with APRA CPS 234 risk governance standards. Risk outputs are bucketed into CRITICAL / HIGH / MEDIUM / LOW bands with calibrated probability scores.

**Layer 2 — RAG Policy Assistant**

FAISS vector store indexed with Sentence-Transformers embeddings over 847 insurance policy documents. LangChain orchestrates multi-turn query routing through Anthropic Claude, returning clause-level cited answers with source attribution and confidence scoring. Policy lookup time reduced by an estimated 70% versus manual document search.

**Layer 3 — Agentic Claims Triage**

Multi-agent tool-calling system that autonomously classifies claim severity, extracts structured metadata, flags anomaly patterns, and generates natural-language adjuster summaries — mirroring the agentic workflows Suncorp has deployed at enterprise scale with measurable operational savings.

**Stack:** Python 3.11 · XGBoost · SHAP · LangChain · FAISS · Sentence-Transformers · Anthropic Claude API · FastAPI · Pydantic v2 · Docker · GitHub Actions · Hugging Face Spaces

### Apex — Revenue Intelligence Platform

<div align="center">

[![Live Dashboard](https://img.shields.io/badge/Live_Dashboard-Visit-D4A853?style=for-the-badge)](https://RameshSTA.github.io/apex)&nbsp;
[![Repository](https://img.shields.io/badge/GitHub-Repository-181717?style=for-the-badge)](https://github.com/RameshSTA/apex)

</div>

<br/>

A production-grade airline revenue-management intelligence platform solving the four highest-leverage operational problems in commercial aviation: demand forecasting, cabin-mix seat optimisation, causal price-elasticity identification, and network-scale RM automation via agentic orchestration. Grounded in Qantas Group's FY2025 operational scale — A$23.8B revenue, 55.9M passengers, 363 aircraft — where a 1% yield improvement on the A$18B passenger revenue base is worth approximately **A$180M annually**.

<br/>

<div align="center">

| Metric | Value | Context |
|:---|:---:|:---|
| LP Revenue Uplift | **7.3%** | scipy HiGHS vs flat-allocation baseline |
| Hybrid Forecast MAPE | **< 14%** | 10 Australian domestic routes, 26-week holdout |
| LP Solve Time | **< 1 ms** | 4-variable · 5-constraint LP via HiGHS backend |
| Agent End-to-End Latency | **< 300 ms** | Plain-language brief → LP → recommendation |

</div>

<br/>

**Forecasting Layer**

Hybrid Holt-Winters + Gradient-Boosted residual model engineered on 30 domain-specific features across 10 Australian domestic routes. Walk-forward cross-validation with bootstrap confidence intervals ensures calibrated uncertainty under event-driven regimes — school-holiday interactions, competitor entry, and the Rex Airlines administration natural experiment. The 0.7-point MAPE improvement over pure Holt-Winters translates to materially lower downstream cabin-allocation error at Qantas operating scale.

**Optimisation Layer**

scipy HiGHS linear programme maximises cabin-mix revenue across four fare classes with access-floor constraints. Dual variable recovery produces mathematically defensible bid prices — replacing static quarterly analyst targets with a continuously optimising, audit-ready allocation engine that scales to 600,000 daily flight-level pricing decisions.

**Causal Inference Layer**

Difference-in-Differences (with parallel-trends placebo validation) quantifies the causal yield impact of the Rex administration and Virgin repricing competitor events. 2SLS instrumental variables using the IATA jet-fuel index corrects OLS simultaneity bias in price-elasticity estimation, with Staiger-Stock F-diagnostic, Hausman endogeneity test, and HC1-robust standard errors — producing elasticity estimates that pricing teams can act on without systematic over-discounting.

**Agentic Layer**

Tool-calling orchestrator that plans and executes forecast → optimise chains from a plain-language route brief, returning a fully auditable recommendation trail with every tool call logged — making the analyst's judgement scalable across the entire network.

**Stack:** Python 3.10+ · Holt-Winters · GradientBoostingRegressor · scikit-learn · scipy HiGHS · NumPy · Difference-in-Differences · 2SLS IV · Chart.js

### FraudSentinel — Proactive Payment Fraud Intelligence

<br/>

Most fraud detection systems react after a transaction is already flagged. FraudSentinel detects risk **72 hours before the financial loss materialises** — built on 590,540 real IEEE-CIS payment transactions with a forward-looking early-warning label engineered to identify entities that will commit fraud within the next three days.

<div align="center">

| Metric | Value | Context |
|:---|:---:|:---|
| Operational Lift | **4.7×** | Over random review queue |

### CLV Long-Term Optimisation — Customer Value Forecasting & Retention

<div align="center">

[![Repository](https://img.shields.io/badge/GitHub-Repository-181717?style=for-the-badge)](https://github.com/RameshSTA/clv-long-term-optimization)

</div>

<br/>

Most retention programmes treat all at-risk customers equally. This system answers a more precise question: *which customers are worth retaining, by how much, and what is the optimal budget allocation to maximise return?* Built on 4,933 customers representing a £12.1M revenue corpus, it combines probabilistic CLV forecasting, machine learning churn classification, and integer-programming budget optimisation into a single automated decision pipeline.

<div align="center">

| Metric | Value | Context |
|:---|:---:|:---|
| Churn Model AUC-ROC | **0.816** | Champion Random Forest, 5-fold CV |
| CLV Forecast Rank | **ρ = 0.57** | Spearman correlation, 180-day holdout |
| Top-Decile Revenue Lift | **6.3×** | vs portfolio mean |
| Net ROI at £200 Budget | **44.5×** | PuLP/CBC optimisation frontier |
| Top 10% Revenue Share | **62%** | Gini = 0.73 |

</div>

<br/>

A BG/NBD + Gamma-Gamma probabilistic engine forecasts 180-day customer revenue. A champion Random Forest (AUC-ROC 0.816, AUC-PR 0.883) identifies churn risk. A PuLP/CBC 0-1 Knapsack allocator returns the optimal retention target set under any spend constraint. A 500-draw Monte Carlo simulation delivers P5–P95 confidence bands for every budget scenario. The fully automated 11-step pipeline runs end-to-end from a single command, tracked in MLflow with GitHub Actions CI across Python 3.9 and 3.11.

**Stack:** BG/NBD · Gamma-Gamma · XGBoost · LightGBM · Random Forest · PuLP/CBC · Monte Carlo · MLflow · GitHub Actions

### FinGraph-ASX — GraphRAG Financial Intelligence Engine

<div align="center">

[![Repository](https://img.shields.io/badge/GitHub-Repository-181717?style=for-the-badge)](https://github.com/RameshSTA/FinGraph-ASX-GraphRAG-Engine)

</div>

<br/>

Financial due diligence on ASX-listed companies is time-consuming and error-prone when done manually across dense annual reports. FinGraph replaces keyword search with an agentic GraphRAG pipeline — combining a Neo4j knowledge graph of extracted financial entities with LLM reasoning — so analysts can ask complex questions in plain English and receive answers grounded in traceable, source-linked evidence from the original filings.

<div align="center">

| Metric | Value | Context |
|:---|:---:|:---|
| Retrieval Latency Reduction | **40%** | vs vector-only semantic search |
| LLM Answer Accuracy | **+15%** | vs baseline, citation-verified |
| Entities Auto-Extracted | **50+** | Per ASX annual report |

</div>

<br/>

A custom NLP extraction pipeline maps 50+ financial entities (companies, executives, subsidiaries, financial metrics, risk factors) into Neo4j. At query time, LlamaIndex traverses the graph rather than performing vector-only search, improving precision on multi-hop financial questions. A citation-style verification layer attaches source sentence, page, and document to every LLM response — eliminating hallucinations for financial analysis where accuracy directly affects investment decisions.

**Stack:** LlamaIndex · Neo4j · Groq · Streamlit · Python
